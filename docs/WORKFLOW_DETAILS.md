### Dependent Workflow Modules (Created through business flow)

| Module | How it's created | Dependencies |
|--------|-----------------|---------------|
| **Service Requests** | Public form OR admin creates from customer context | Customer + Vehicle must exist |
| **Appointments** | Created from Service Request OR directly by admin | Customer + Vehicle (+ optional SR) |
| **Job Cards** | Created from Appointment/Service Request | Customer + Vehicle + (optional Appointment + SR) |
| **Invoices** | Created from completed Job Card | Customer + Vehicle + Job Card |
| **Payments** | Recorded against an Invoice | Invoice must exist |
| **Stock Movements** | Created when inventory is adjusted | Inventory Item must exist |
| **Worker Assignments** | Assigned to a Job Card | Worker + Job Card must exist |
| **Worker Leave** | Created for a Worker | Worker must exist |
| **Notifications** | Auto-generated by system events | Template + recipient |
| **Activity Logs** | Auto-generated on every mutation | — (system-managed) |

## Primary Business Workflow

```
┌─────────────────────────────────────────────────────────────────────┐
│                    VEHICLE SERVICING LIFECYCLE                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  1. SERVICE REQUEST                                                 │
│     └─ Customer submits via public form OR admin creates            │
│        Status: SUBMITTED → UNDER_REVIEW → APPOINTMENT_PENDING       │
│                                                                     │
│  2. APPOINTMENT                                                     │
│     └─ Admin confirms appointment from service request              │
│        Status: REQUESTED → CONFIRMED → CHECKED_IN → COMPLETED       │
│                                                                     │
│  3. JOB CARD                                                        │
│     └─ Created when vehicle arrives (from appointment/SR)           │
│        Status: CREATED → UNDER_INSPECTION → ESTIMATE_PREPARED       │
│              → AWAITING_CUSTOMER_APPROVAL → APPROVED                │
│              → WORK_IN_PROGRESS → QUALITY_CHECK                     │
│              → READY_FOR_DELIVERY → DELIVERED                       │
│                                                                     │
│  4. INVOICE                                                         │
│     └─ Generated from job card when work is complete                │
│        Status: DRAFT → FINALIZED                                    │
│                                                                     │
│  5. PAYMENT                                                         │
│     └─ Recorded against finalized invoice                           │
│        Invoice Payment Status: UNPAID → PARTIALLY_PAID → PAID       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Module Details

### 1. Service Requests

**Type:** Dependent (entry point of workflow)

**Creation methods:**
- Public form at `/book-service` → `POST /api/public/service-requests`
- Admin cannot directly create from admin panel (no "Create" button)
- Admin can only view and update status

**Status flow:**
```
SUBMITTED → UNDER_REVIEW → APPOINTMENT_PENDING → APPOINTMENT_CONFIRMED → CONVERTED_TO_JOB
                                                                        → CANCELLED
                                                                        → CLOSED
```

**Admin actions:** View list, view detail, update status, add notes

---

### 2. Appointments

**Type:** Dependent (created from SR or directly)

**Creation:** Admin creates from appointment page with customer + vehicle + date/slot

**Status flow:**
```
REQUESTED → PENDING_REVIEW → CONFIRMED → CHECKED_IN → COMPLETED
                            → RESCHEDULED (back to CONFIRMED)
                            → CANCELLED
                            → NO_SHOW
```

**Admin actions:** Create, confirm, reschedule, cancel, check-in, mark no-show

---

### 3. Job Cards

**Type:** Dependent (created from appointment/SR)

**Creation:** Admin creates with customer + vehicle + issue summary. Optionally links to appointment and service request.

**Status flow:**
```
CREATED → UNDER_INSPECTION → ESTIMATE_PREPARED → AWAITING_CUSTOMER_APPROVAL
       → APPROVED → PARTS_PENDING → WORK_IN_PROGRESS → QUALITY_CHECK
       → READY_FOR_DELIVERY → DELIVERED → CLOSED
       (any stage) → CANCELLED
```

**Sub-entities:**
- Tasks (create, update status, assign worker)
- Parts (add from inventory, track consumption)
- Worker Assignments (assign/unassign workers)

**Admin actions:** Create, update status, add diagnosis, manage tasks/parts/workers

---

### 4. Invoices

**Type:** Dependent (created from job card)

**Creation:** Admin creates with customer + vehicle + job card + line items

**Line item types:** PART, LABOR, CUSTOM_CHARGE, DISCOUNT_ADJUSTMENT

**Calculations:**
- `lineTotal = quantity × unitPrice + taxAmount`
- `taxAmount = quantity × unitPrice × (taxRate / 100)`
- `grandTotal = subtotal + taxTotal - discountAmount`

**Status flow:**
```
DRAFT → FINALIZED → (payments recorded)
     → CANCELLED
```

**Payment status:** `UNPAID → PARTIALLY_PAID → PAID`

**Admin actions:** Create, view, finalize, record payments

---

### 5. Payments

**Type:** Dependent (recorded against invoice)

**Creation:** Admin records payment against a finalized invoice

**Fields:** amount, paymentMode (CASH/CARD/UPI/BANK_TRANSFER/CHEQUE), date, reference

**Auto-updates:** Invoice `amountPaid`, `amountDue`, `paymentStatus`

---

### 6. Inventory

**Type:** CRUD (items, categories, suppliers) + Dependent (stock movements)

**Stock Movement types:**
```
STOCK_IN, STOCK_OUT, ADJUSTMENT_INCREASE, ADJUSTMENT_DECREASE,
RESERVED, RELEASED, CONSUMED, RETURNED
```

**Auto-calculation:** Movement updates `quantityInStock` on the item

---

### 7. Expenses

**Type:** CRUD

**Creation:** Admin creates with date, category, title, amount, payment mode

**Admin actions:** Create, view, delete

---

### 8. Workers

**Type:** CRUD + sub-entities

**Sub-entities:**
- Leave requests (create, approve)
- Schedule view (assignments + leaves)

**Status:** ACTIVE, INACTIVE, ON_LEAVE

---

### 9. Notifications

**Type:** System-generated (view only in admin)

**Channels:** WHATSAPP, EMAIL

**Status flow:** `QUEUED → PROCESSING → SENT → DELIVERED` or `→ FAILED → DEAD_LETTER`

**Admin actions:** View list, retry failed, manage templates

---

### 10. Reports & Dashboard

**Type:** Read-only aggregations

**Dashboard metrics:**
- Today's appointments
- Pending service requests
- Active job cards
- Unpaid invoices
- Today's revenue

**Report types:** Revenue, Appointments, Jobs, Inventory, Workers, Expenses

---

### 11. Settings

**Type:** CRUD (key-value)

**Categories:**
- Business info (name, phone, email, address, GST)
- Invoice config (prefix, footer)
- Notification config (WhatsApp/email enabled)
- Business hours (slot rules, holidays)
- Integrations

---

### 12. Activity Logs

**Type:** Read-only (auto-generated)

**Captured on:** Every create/update/delete across all modules

**Fields:** entityType, entityId, action, previousValue, newValue, actorType, actorId, timestamp
